# นโยบายความเป็นส่วนตัว — Meetily Thai รุ่น PC

อัปเดตล่าสุด: 20 กรกฎาคม 2569

## หลักการ

Meetily Thai รุ่น PC ออกแบบให้บันทึกเสียง ถอดเสียง และจัดเก็บข้อมูลการประชุมภายในเครื่องเป็นค่าเริ่มต้น ผู้ใช้และหน่วยงานเป็นผู้ควบคุมไฟล์เสียง บทถอดเสียง สรุป และเอกสารที่สร้างขึ้น

## ข้อมูลที่จัดเก็บในเครื่อง

- ไฟล์เสียงการประชุม
- บทถอดเสียงและข้อมูลช่วงเวลา
- สรุป บันทึก และข่าวการประชุม
- ค่าตั้งโปรแกรมและโมเดลที่ดาวน์โหลด
- API key หรือ GAS shared secret ซึ่งเก็บผ่าน Credential Manager/Keyring ของระบบปฏิบัติการ

## การส่งข้อมูลออกจากเครื่อง

รุ่นนี้ปิด Telemetry/Analytics และ Auto Update อย่างถาวร จึงไม่มีการส่งสถิติการใช้งานไปยังเจ้าของโครงการเดิม

การเชื่อมต่อภายนอกจะเกิดขึ้นเมื่อผู้ใช้เลือกใช้ความสามารถต่อไปนี้เท่านั้น:

1. ดาวน์โหลดโมเดล Whisper/Parakeet หรือโมเดลสรุปภายในเครื่อง
2. ใช้ Cloud LLM เช่น OpenAI, Anthropic, Groq หรือ OpenRouter หลังผู้ใช้ตั้งค่าเอง
3. ใช้ Ollama หรือ Custom LLM ตาม endpoint ที่ผู้ใช้กำหนดและผ่านนโยบายความปลอดภัย
4. ส่งเอกสารไปยัง Google Apps Script หลังผู้ใช้ตั้งค่า URL/secret และสั่งส่ง
5. เปิดหน้าดาวน์โหลด Ollama หรือเอกสาร Google ที่โปรแกรมอนุญาต

เมื่อเปิด Confidential Mode ซึ่งเป็นค่าเริ่มต้น โปรแกรมจะอนุญาตเฉพาะ Local Whisper และผู้ให้บริการสรุปภายในเครื่อง/localhost ตามนโยบายที่กำหนด

## สิทธิ์และหน้าที่ของผู้ควบคุมข้อมูล

หน่วยงานควรกำหนดอย่างน้อย:

- ผู้มีสิทธิ์บันทึกและเปิดดูข้อมูล
- ระยะเวลาเก็บไฟล์เสียงและบทถอดเสียง
- วิธีสำรอง กู้คืน ส่งออก และทำลายข้อมูล
- ขั้นตอนแจ้งและขอความยินยอมจากผู้เข้าร่วมประชุม
- ข้อจำกัดการใช้ Cloud LLM หรือ Google Apps Script กับข้อมูลลับ

## การลบข้อมูล

ผู้ใช้สามารถลบข้อมูลการประชุมและไฟล์ที่เกี่ยวข้องจากโปรแกรมหรือโฟลเดอร์จัดเก็บในเครื่อง การถอนการติดตั้งโปรแกรมอาจไม่ลบข้อมูลผู้ใช้โดยอัตโนมัติ จึงควรตรวจโฟลเดอร์ข้อมูลแอปและ Credential Manager ตามนโยบายของหน่วยงาน

## ข้อจำกัด

ความปลอดภัยของข้อมูลยังขึ้นกับบัญชี Windows สิทธิ์โฟลเดอร์ การเข้ารหัสดิสก์ การสำรองข้อมูล และการตั้งค่าเครือข่ายของเครื่องปลายทาง
